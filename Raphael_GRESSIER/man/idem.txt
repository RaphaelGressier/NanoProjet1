NOM
idem.sh
SYNOPSISidem.sh [OPTION...] [chemin...] [expression]
DESCRIPTIONidem.sh écrit sur stdout tous les noms complets (répertoires et nom de base du fichier) des fichiers correspondant à expression dans les arborescences de répertoires de chaque chemin mentionné.
OPTIONS-h, --helpafficher l’aide en ligne et quitter-v, --verboseexpliquer ce qui est fait
EXPRESSIONS
-iname motifSemblable à -name, mais sans différencier les majuscules et les minuscules. Par exemple les motifs «to* » et « T??? » correspondent aux noms de fichiers « Toto », « TOTO », « toto », « tOto », etc.
-name motifLe nom de base du fichier cherché (sans les répertoires du chemin d’accès) correspond à motif. Un motif est soit un nom simple, soit une expression rationnelle compatible avec grep.idem.sh – Chercher des fichiers identiques dans des arborescences
DUT S1T : nano projet Semaine du 12 novembre 2012 -size nLe fichier cherché est de taille n octets-includes fichierLe fichier cherché est de type texte et contient tous les mots écrits dans fichier (un mot par ligne)VARIABLES D'ENVIRONNEMENTIGNORE_CASE Si elle contient une chaîne de caractères non vide, alors -name est forcé à -inameEXEMPLESidem.sh ~/ASR3 ~/TMP/ASR3 /SupportCours/S1T -name paste.shCherche tous les fichiers de nom de base paste.sh dans les arborescences de racine ~/ASR3, ~/TMP/ASR3 et /SupportCours/S1T.idem.sh ASR3/TP -includes mots.txtCherche tous les fichiers dans ./ASR3/TP qui contiennent les mots du fichier mots.txt. Exemple de fichier mots.txt (sur deux lignes) qui permettrait de retrouver tous les scripts bash qui utilisent la commande grep:/bin/bashgrepVALEURS DE RETOURidem.sh termine en renvoyant la valeur de retour 0 si le traitement a été effectué sans erreur.
